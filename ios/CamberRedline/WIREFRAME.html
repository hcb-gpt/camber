Data Flow Architecture

The iOS app reads contact lists via PostgREST (direct Supabase view query) and thread data via edge function. Grades are written via POST to the same edge function. Realtime subscriptions push grade changes back to the app.

Contact List Data Path

  • Source: redline_contacts SQL view (aggregated from interactions + journal_claims + sms_messages + claim_grades)
  • Method: PostgREST via supabase-swift: client.from("redline_contacts").select().order("last_activity")
  • Fields: contact_id, contact_name, contact_phone, call_count, sms_count, claim_count, ungraded_count, last_activity, last_snippet, last_direction, last_interaction_type
  • Realtime: Subscribes to INSERT on interactions table; triggers full reload of contacts list

Thread Data Path

  • Source: Edge function redline-thread
  • Request: GET ?contact_id=UUID&limit=100
  • Response: { ok, contact: {id,name,phone}, thread: [{type,event_at,direction,...}], pagination: {total,limit,offset} }
  • Thread items: Polymorphic — type:"call" has spans+claims, type:"sms" has content
  • Sorting: Client-side by eventAt ascending (oldest first, newest at bottom)

Grade Write Path

  • Endpoint: POST /redline-thread
  • Payload: { claim_id, grade, correction_text?, graded_by: "ios_reviewer" }
  • Grade values: "confirm" | "reject" | "correct"
  • After write: Full thread reload + Realtime subscription merges grade in-place

Realtime Subscription (claim_grades)

  • Channel: claim-grades-{contactId}
  • Events: INSERT + UPDATE on claim_grades table
  • Merge logic: Walks threadItems -> calls -> spans -> claims, finds matching claim_id, updates grade/correctionText/gradedBy in-place
  • Lifecycle: Starts on thread load, stops on view disappear

Known Limitations

Data Gaps Critical

  • Edge function vs. view mismatch: The edge function uses source_span_id join (only 21% of claims visible for high-volume contacts like Zack). The redline_thread SQL view uses the call_id path which returns all 663 claims. The iOS app currently hits the edge function, not the view directly.
  • Span coverage: Only 32/168 Zack interactions have pipeline-generated spans (19%). Jan 28 - Feb 21 = processed range. Older calls have no spans/claims.
  • Char pointers broken: 99%+ of journal_claims have misaligned char_start/char_end after transcript reprocessing. Sub-span highlighting not viable in v1.
  • SMS: No pipeline integration. 120 raw messages displayed as-is. No claims extracted from SMS content.

UX Limitations Moderate

  • No search or filter. Contact list is a flat scroll ordered by last_activity. No way to search by name or filter by ungraded.
  • No pagination. Thread loads up to limit=100 items. Contacts with 168+ interactions may be truncated.
  • Pull-to-refresh only. No Realtime push for new calls/SMS. User must manually pull to see new interactions.
  • Single grader identity: All grades stamped as "ios_reviewer". No user auth or multi-user support.
  • No undo for grades. Once a claim is graded, it can be re-graded (upsert) but there's no explicit undo UI.
  • Transcript always collapsed. DisclosureGroup defaults closed. No way to expand all.

Security / Auth Critical

  • Anon key hardcoded in SupabaseService.swift. No user authentication. Relies entirely on RLS policies.
  • Edge function verify_jwt=false. Anyone with the URL can read threads and write grades.
  • No rate limiting on grade submissions or thread fetches.
  • Deploy method: Xcode direct install (free Apple ID, 7-day re-sign). Not on TestFlight yet ($99/yr).

Architecture Debt Moderate

  • Dual data paths: Contacts via PostgREST, thread via edge function. Should unify.
  • Full thread reload after grading. The grade POST triggers a full thread re-fetch. Realtime merge exists but the reload still fires.
  • No offline support. App requires network for all operations. No local cache or CoreData.
  • ViewModels shared at app level. ThreadViewModel is shared across navigation, which means navigating to a different contact overwrites the previous thread state.

Improvement Suggestions

Priority 1 — Data Completeness High Impact

  • Switch thread source from edge function to PostgREST view. The redline_thread view returns 6x more claims (call_id join vs source_span_id). This is the single biggest improvement.
  • Increase limit or add true pagination. Use offset/limit with "load more" button or infinite scroll to handle contacts with 100+ interactions.
  • Backfill older interactions through the pipeline to generate spans + claims for pre-Jan 28 calls.

Priority 2 — UX Enhancements Quick Wins

  • Add search bar to ContactListView (.searchable modifier). Filter by contact name.
  • Add "ungraded first" sort option or a filter toggle to show only contacts with ungraded claims.
  • Batch grading: Allow multi-select on claims to confirm/reject several at once.
  • Grade summary stats at top of thread: "23 claims: 15 confirmed, 3 rejected, 2 corrected, 3 ungraded".
  • Haptic feedback: Add UINotificationFeedbackGenerator(.success) on grade confirm, .error on reject. Currently only contextMenu has system haptics.

Priority 3 — Security Hardening Required for Production

  • Enable verify_jwt on edge function. Add user auth (even simple PIN/passcode) before production use.
  • Move anon key to Xcode build config or Info.plist instead of hardcoded string literal.
  • Add grader identity: Replace hardcoded "ios_reviewer" with device name or authenticated user ID.
  • Rate limit grade submissions at the edge function level.

Priority 4 — Architecture Medium-Term

  • Unify data path: Move thread fetching to PostgREST (same as contacts) to eliminate the edge function dependency for reads.
  • Per-contact ThreadViewModel: Create a new ViewModel instance per contact to preserve state when navigating between contacts.
  • Add Realtime for new interactions in thread view (not just contact list) so new calls/SMS appear without pull-to-refresh.
  • Offline cache: Add SwiftData or lightweight JSON cache for recently viewed threads.
  • TestFlight deployment: Set up Apple Developer account ($99/yr) for stable distribution without 7-day re-sign.
